# STEP049B — LANDING IMPLEMENTATION

## Goal

Implement the production landing uplift for Intro Deck based on the STEP049A landing contract.

## Scope

- rebuild the public landing structure
- strengthen hero, CTA hierarchy, and section rhythm
- add who-it's-for, how-it-works, what's-inside, why-it's-easier, product preview, FAQ, and final CTA sections
- improve legal-page navigation consistency
- keep the implementation narrow: static HTML + CSS only

## Out of scope

- OG/social asset generation
- metadata expansion beyond basic title/description
- multilingual landing variants
- heavy animation or JavaScript-driven marketing effects

## Product truth

- Intro Deck remains a Telegram-native private professional directory and contact workflow
- LinkedIn stays the identity layer, not a broad profile import or scraping surface
- The landing should act as a clean entry page, not a multi-page marketing site

## Acceptance

- Hero clearly explains the product and primary action
- Legal links no longer compete with the hero CTA
- Landing includes the agreed section map from STEP049A
- Copy stays direct, product-grade, and free of generic SaaS filler
- Mobile layout remains readable and button hierarchy stays clear
